## 1. 프로젝트 개요

### 목적

인증된 사용자가 개인별로 할 일을 체계적으로 관리할 수 있는 웹 애플리케이션을 제공한다. 시작일과 종료일 기반의 일정 관리 기능을 통해 기존 할 일 관리 도구의 기능적 한계를 보완한다.

### 범위

- 사용자 인증 (회원가입 / 로그인 / 프로필 수정)
- 카테고리 기반 할 일 등록 / 수정 / 삭제
- 시작일 · 종료일 지정 및 캘린더 UI 제공
- 상태 및 카테고리 기반 할 일 목록 필터링

## 3. 도메인 규칙

### 인증 규칙

- R-AUTH-01: 회원가입 없이는 앱을 사용할 수 없다.
- R-AUTH-02: 로그인한 사용자만 할 일 등록 / 수정 / 삭제가 가능하다.
- R-AUTH-03: 사용자는 자신의 프로필(이름, 비밀번호)을 수정할 수 있다. 이메일은 변경할 수 없다.
- R-AUTH-04: 이메일은 시스템 내 고유해야 하며, 중복 이메일로 가입할 수 없다.
- R-AUTH-05: 회원 탈퇴 시 해당 사용자의 모든 카테고리와 할 일 데이터는 함께 삭제된다(cascade delete).
- R-AUTH-06: 비밀번호는 8자 이상 128자 이하여야 하며, 영문자와 숫자를 각각 1자 이상 포함해야 한다.

### 카테고리 규칙

- R-CAT-01: 할 일 등록 시 카테고리를 지정하지 않으면 `기본` 카테고리가 자동 적용된다.
- R-CAT-02: `기본` 카테고리는 사용자 가입 시 자동 생성되며 삭제할 수 없다.
- R-CAT-03: 사용자는 자신이 소유한 카테고리만 조회 및 사용할 수 있다.
- R-CAT-04: 사용자 정의 카테고리 삭제 시, 해당 카테고리에 속한 할 일은 `기본` 카테고리로 자동 이동된다.
- R-CAT-05: 카테고리명은 1자 이상 30자 이하여야 하며, 동일 사용자 내에서 중복된 이름을 가질 수 없다. 이름 중복 비교는 대소문자를 구분하지 않는다 (예: "Work"와 "work"는 동일 이름으로 처리).

### 할 일 규칙

- R-TODO-01: 사용자는 자신이 등록한 할 일만 조회 / 수정 / 삭제할 수 있다.
- R-TODO-02: 종료일자는 시작일자보다 이전일 수 없다.
- R-TODO-03: 시작일자와 종료일자는 선택 사항이다.
- R-TODO-04: 완료 상태의 할 일도 수정 및 삭제가 가능하다.
- R-TODO-05: 할 일 제목(title)은 필수이며, 1자 이상 100자 이하여야 한다.
- R-TODO-06: 할 일 상세 내용(description)은 선택이며, 최대 1000자까지 입력 가능하다.

### 상태 전이 규칙

- R-STA-01: 할 일은 등록 시 기본값이며, `IN_PROGRESS` 또는 `OVERDUE` 조건을 충족하지 않는 한 `NOT_STARTED` 상태로 유지된다.
- R-STA-02: 오늘 날짜가 시작일자 이후이고 종료일자 이전이면 `IN_PROGRESS` 상태로 간주한다.
- R-STA-03: 사용자가 명시적으로 완료 처리한 경우에만 `DONE` 상태가 된다.
- R-STA-04: 종료일자가 오늘 날짜보다 이전이고 완료 처리되지 않은 경우 `OVERDUE` 상태로 간주한다.

> 주석: `NOT_STARTED`, `IN_PROGRESS`, `OVERDUE` 상태는 시스템이 런타임에 날짜 기준으로 계산하며, DB에는 사용자 액션에 의한 상태(DONE)만 저장한다.

### 설정 규칙

- R-SET-01: 사용자는 UI 테마를 `LIGHT` 또는 `DARK`로 설정할 수 있으며, 설정값은 DB에 저장되어 재로그인 후에도 유지된다.
- R-SET-02: 테마 기본값은 `LIGHT`이며, 신규 가입 시 자동 적용된다.
- R-SET-03: 사용자는 앱 언어를 한국어(`ko`) 또는 영어(`en`)로 설정할 수 있으며, 설정값은 DB에 저장되어 재로그인 후에도 유지된다.
- R-SET-04: 언어 기본값은 한국어(`ko`)이며, 신규 가입 시 자동 적용된다.
- R-SET-05: 비인증 화면(로그인, 회원가입)은 브라우저 언어 설정을 따르되, 지원하지 않는 언어일 경우 한국어(`ko`)를 기본으로 표시한다. 브라우저 언어 코드가 지역 포함 형식(`ko-KR`, `en-US` 등)인 경우 언어 코드 앞 두 자리(`ko`, `en`)를 기준으로 판단한다.

## 5. 할 일 상태 정의

| 상태 | 코드 | 진입 조건 |
|------|------|-----------|
| 미시작 | `NOT_STARTED` | 등록 시 기본값이며, `IN_PROGRESS` 또는 `OVERDUE` 조건을 충족하지 않는 한 유지된다 |
| 진행 중 | `IN_PROGRESS` | 오늘 날짜 >= 시작일자 이고, 오늘 날짜 <= 종료일자 |
| 완료 | `DONE` | 사용자가 명시적으로 완료 처리한 경우 |
| 기한 초과 | `OVERDUE` | 오늘 날짜 > 종료일자 이고, 상태가 `완료`가 아닌 경우 |

> 주석: `NOT_STARTED`, `IN_PROGRESS`, `OVERDUE` 상태는 시스템이 런타임에 날짜 기준으로 계산하며, DB에는 사용자 액션에 의한 상태(DONE)만 저장한다.

### 상태 전이 다이어그램

```
[등록]
  |
  v
NOT_STARTED
  |          \
  |           \-- (오늘 >= 시작일)
  |                     |
  |                     v
  |               IN_PROGRESS
  |                     |
  |    (오늘 > 종료일)  |    (사용자 완료 처리)
  |          |           |           |
  |          v           v           v
  +------> OVERDUE      DONE <------+
```

> 참고: `OVERDUE`는 시스템이 날짜 기준으로 자동 판단하는 상태이다. `DONE`은 사용자 액션에 의해서만 전이된다. `DONE` 상태에서는 `OVERDUE`로 전이되지 않는다.

### 날짜 조합별 엣지 케이스

아래 표는 시작일(start_date)과 종료일(end_date)의 설정 조합에 따라 각 상태 조건이 어떻게 적용되는지를 정의한다.

| 케이스 | start_date | end_date | NOT_STARTED 조건 | IN_PROGRESS 조건 | OVERDUE 조건 |
|--------|-----------|----------|-----------------|-----------------|-------------|
| 둘 다 없음 | null | null | 항상 NOT_STARTED (날짜 조건 없음) | 해당 없음 | 해당 없음 |
| 시작일만 있음 | 설정됨 | null | 오늘 < start_date | 오늘 >= start_date | 해당 없음 (종료일 없으면 OVERDUE 불가) |
| 종료일만 있음 | null | 설정됨 | 오늘 <= end_date | 해당 없음 (시작일 없으면 IN_PROGRESS 불가) | 오늘 > end_date |
| 둘 다 있음 | 설정됨 | 설정됨 | 오늘 < start_date | start_date <= 오늘 <= end_date | 오늘 > end_date |
| 시작일 = 종료일 | D | D | 오늘 < D | 오늘 = D | 오늘 > D |

**케이스별 추가 설명**

- **시작일 = 종료일**: 해당 날짜 당일에만 `IN_PROGRESS`로 계산되며, 그 다음날부터는 즉시 `OVERDUE`로 전이된다.
- **시작일만 있고 종료일이 없는 경우**: 시작일 이후에는 `IN_PROGRESS`로 유지되며, 종료일이 없으므로 `OVERDUE`로 전이되지 않는다.
- **종료일만 있고 시작일이 없는 경우**: 시작일 조건을 판단할 수 없으므로 `IN_PROGRESS`로 전이되지 않는다. 오늘이 종료일을 초과하면 `OVERDUE`로 전이된다.
- **시작일도 종료일도 없는 경우**: 날짜 기반 상태 전이가 발생하지 않으므로 `DONE` 처리 전까지 영구적으로 `NOT_STARTED`로 유지된다.
- **DONE 상태에서 미완료로 되돌리기**: `DONE` 상태는 사용자가 미완료(undo) 처리를 통해 이전 상태로 되돌릴 수 있다. 되돌린 후의 상태는 현재 날짜와 날짜 조합에 따라 시스템이 `NOT_STARTED`, `IN_PROGRESS`, `OVERDUE` 중 하나로 재계산한다.
