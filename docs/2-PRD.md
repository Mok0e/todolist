## 2. 제품 개요

### 목적

인증된 사용자가 카테고리와 날짜 기반으로 할 일을 체계적으로 관리할 수 있는 웹 애플리케이션을 제공한다.

### 문제 정의

- 기존 메모 도구나 단순 체크리스트는 시작일·종료일 기반의 일정 관리를 지원하지 않는다.
- 카테고리 분류 없이 할 일이 쌓이면 우선순위 파악이 어렵다.
- 기한 초과 항목을 별도로 추적하는 기능이 부재하여 업무·학습 관리의 사각지대가 발생한다.

### 제품 비전

"학생과 직장인이 시작일·종료일과 카테고리로 할 일을 구조적으로 관리하고, 기한 초과를 놓치지 않도록 돕는 경량 웹 투두 서비스"

## 4. 타겟 사용자

### 세그먼트 1: 학생

- 학기 중 과제·시험 일정을 날짜 기반으로 등록하고 싶은 사용자
- 과목별 카테고리 분류로 할 일을 정리하려는 니즈
- 기한 초과 항목을 빠르게 파악하여 우선순위를 재조정하는 패턴

### 세그먼트 2: 직장인

- 업무 태스크를 프로젝트·업무 유형별 카테고리로 관리하려는 사용자
- 시작일·종료일이 명확한 업무를 캘린더 관점에서 시각화하려는 니즈
- 완료/미완료 항목을 필터링하여 진행 상황을 빠르게 파악하는 패턴

## 5. 기능 범위

### In-scope (이번 릴리즈 포함)

- 회원가입 / 로그인 / 로그아웃 / 프로필 수정 / 회원 탈퇴
- 할 일 CRUD (등록 / 수정 / 삭제 / 목록 조회)
- 할 일 완료 처리 및 완료 취소
- 할 일 상태 자동 계산 (NOT_STARTED / IN_PROGRESS / OVERDUE / DONE)
- 카테고리 CRUD (생성 / 수정 / 삭제, 기본 카테고리 자동 생성 포함)
- 상태 · 카테고리 기반 필터링
- 반응형 웹 UI

### Out-of-scope (이번 릴리즈 제외)

- 모바일 네이티브 앱 (iOS / Android)
- 소셜 로그인 (Google / Kakao 등 OAuth)
- 할 일 공유 / 협업 기능
- 알림 / 푸시 알림
- 파일 첨부
- 할 일 순서 드래그 앤 드롭 정렬
- 접근성 (WCAG) 대응

### UC-14: 다국어 지원 (v2)

> **릴리즈**: v2

**연관 규칙:** R-AUTH-02, R-AUTH-03, R-SET-03, R-SET-04, R-SET-05

**개요**
인증된 사용자가 앱 전체 언어를 한국어(ko) 또는 영어(en)로 변경할 수 있다. 선택한 언어는 User 엔티티의 `language` 속성에 저장되어 재로그인 후에도 유지된다.

**Acceptance Criteria**

| #        | 시나리오                  | 조건                                                       | 기대 결과                                               |
| -------- | ------------------------- | ---------------------------------------------------------- | ------------------------------------------------------- |
| AC-14-01 | 언어 변경 — 영어          | 설정 화면에서 영어(en) 선택 후 저장                        | 앱 전체 UI가 영어로 전환, `language = en` DB 저장       |
| AC-14-02 | 언어 변경 — 한국어        | 설정 화면에서 한국어(ko) 선택 후 저장                      | 앱 전체 UI가 한국어로 전환, `language = ko` DB 저장     |
| AC-14-03 | 재로그인 후 언어 유지     | 영어로 설정 후 로그아웃 → 재로그인                         | 로그인 후 영어 UI 유지                                  |
| AC-14-04 | 기본 언어                 | 신규 가입 사용자 첫 로그인                                 | 한국어(ko)로 표시                                       |
| AC-14-05 | 비인증 화면 — 지원 언어   | 브라우저 언어가 `ko` 또는 `en`(지역 포함 코드 포함)인 경우 | 해당 언어로 표시 (예: `ko-KR` → 한국어, `en-US` → 영어) |
| AC-14-06 | 비인증 화면 — 미지원 언어 | 브라우저 언어가 `ko`, `en` 이외인 경우                     | 한국어(`ko`)로 기본 표시 (R-SET-05)                     |

- 날짜 입력은 date picker 컴포넌트 사용 (브라우저 native 또는 경량 라이브러리)
- 종료일 picker는 시작일 이후 날짜만 선택 가능하도록 제한
- 할 일 목록에서 기한 초과(OVERDUE) 항목은 시각적으로 구분 (예: 붉은 색상 표시)

### 필터 UI 방향

- 상태 필터와 카테고리 필터는 동시 적용 가능
- 필터 초기화 버튼 제공
- 적용 중인 필터는 시각적으로 활성 상태 표시

### v2 UI 요구사항

**테마 (Dark/Light Mode)**

- CSS 변수 또는 Tailwind dark 클래스 기반 테마 전환
- 테마 전환은 페이지 새로고침 없이 즉시 적용 (Optimistic Update)
- 저장 실패 시 이전 테마로 자동 롤백

**다국어 (i18n)**

- i18n 라이브러리 사용 (react-i18next 권장)
- 지원 언어: 한국어(ko), 영어(en)
- 번역 키는 컴포넌트 내 하드코딩 금지, 별도 locale 파일 관리

| R-04 | 1,000명 동시 접속 — 단일 서버 성능 한계            | 중간   | 낮음   | Connection pool 설정 최적화, 필요 시 인덱스 추가            |
| R-05 | TanStack Query 캐시와 서버 상태 불일치             | 낮음   | 중간   | mutation 후 관련 query invalidation 패턴 일관 적용          |
| R-06 | 상태 자동 계산(런타임) — 프론트/백엔드 로직 불일치 | 중간   | 중간   | 상태 계산 로직 백엔드 단일화, 프론트는 API 응답값 사용      |
